The candidate in the annotation-r2 worktree uses a sequence-indexed `xizi_dll_open` predicate for the interior of the circular doubly-linked path. The C precondition owns the removed node's two fields and the two boundary fields (`next.prev` and `prev.next`) explicitly; the postcondition updates those boundary values and owns the removed node as self-linked. This avoids overlapping ownership when the successor and predecessor are the same sentinel.

The first direct symbolic-execution attempt failed because the original opaque segment predicate hid `next.prev`. The repaired candidate passed three complete canonical symexec runs, the case library fixed Coq compile gate, and a full goal-check compile. The round is nevertheless blocked because this agent environment exposed neither a `qcp-mcp` tool nor a local `qcp-mcp` executable. Under annotation-checking, direct symexec cannot substitute for that requirement. The JSON handoff, current files, and controller versions win on conflict.
