# 常见失败信号

## 典型信号

- 后条件要求的数据结构形状在前条件中根本不存在。
- 前条件缺少证明分支条件所需的纯命题。
- witness 需要的抽象状态与当前 annotation 中维护的状态不一致。
- 修改了 annotation，但刷新后的 VC 与旧版本几乎完全一样，说明可能没有真正重新生成。
- refinement 目标中 `safeExec` 的程序状态对不上，通常不是 tactic 问题，而是抽象状态或中间断言问题。

## 处理建议

- 先确认当前 `goal.v` 是否确实来自当前 C 文件。
- 再检查 invariant 是否覆盖了循环体出口和下一轮入口都需要的信息。
- 如果目标要求某个列表、树或数组关系成立，但前条件没有足够结构资源，优先回看 annotation 设计。
