322. Coin Change

URL: https://leetcode.com/problems/coin-change/

This verification case checks the dynamic-programming reachability core of the
coin-change problem: given positive coin denominations and a target amount, the
function returns the greatest value not exceeding the target that can be formed
using the denominations.  The final functional specification should be phrased
with MaxMinLib as the maximum reachable amount, and the Rocq library should
define the DP state transition used by the C implementation.
